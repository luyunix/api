# Faber API —— 领域层（Domain）：工具系统

工具系统是 Agent 的"手脚"。没有工具，LLM 只能"空谈"；有了工具，Agent 才能真正改变现实世界（至少在沙箱中）。

Faber 的工具系统设计非常优雅：通过**装饰器自动注册**、**反射自动生成 Schema**、**协议抽象统一调用**。

---

## 1. 设计哲学

### 1.1 核心设计原则

1. **声明式**：开发者只需写普通 Python 方法 + `@tool` 装饰器，自动获得 OpenAI Function Calling Schema
2. **类型安全**：工具参数通过 Pydantic 校验，LLM 的幻觉参数会被过滤
3. **统一接口**：所有工具通过 `BaseTool.invoke(tool_name, **kwargs)` 调用，调用方无需了解实现
4. **可扩展**：新增工具只需新增一个方法，零配置注册

### 1.2 工具分类

| 工具集 | 类名 | 功能 |
|--------|------|------|
| 文件工具 | `FileTool` | 读写、搜索、替换、列出文件 |
| Shell 工具 | `ShellTool` | 执行命令、进程管理 |
| 浏览器工具 | `BrowserTool` | 导航、点击、输入、截图、执行 JS |
| 搜索工具 | `SearchTool` | 全网搜索引擎 |
| 消息工具 | `MessageTool` | 通知用户、询问用户 |
| MCP 工具 | `MCPTool` | 调用外部 MCP 服务 |
| A2A 工具 | `A2ATool` | 调用远程 Agent |

## 2. BaseTool 与 @tool 装饰器

### 2.1 @tool 装饰器

```python
def tool(
    name: str,                    # 工具唯一标识（OpenAI function name）
    description: str,             # 工具描述（给 LLM 看的）
    parameters: Dict[str, Dict],  # 参数 Schema
    required: List[str],          # 必填参数
) -> Callable:
    def decorator(func):
        tool_schema = {
            "type": "function",
            "function": {
                "name": name,
                "description": description,
                "parameters": {
                    "type": "object",
                    "properties": parameters,
                    "required": required,
                }
            }
        }
        func._tool_name = name
        func._tool_description = description
        func._tool_schema = tool_schema
        return func
    return decorator
```

**这是整个工具系统的魔法所在**。装饰器将方法的元数据绑定到方法对象上，后续通过反射读取。

### 2.2 BaseTool 基类

```python
class BaseTool:
    name: str = ""  # 工具集名字（如 "browser", "shell"）
    
    def __init__(self):
        self._tools_cache = None
    
    def get_tools(self) -> List[Dict[str, Any]]:
        """获取该工具集下所有工具的 OpenAI Schema"""
        if self._tools_cache is not None:
            return self._tools_cache
        
        tools = []
        for _, method in inspect.getmembers(self, inspect.ismethod):
            if hasattr(method, "_tool_schema"):
                tools.append(getattr(method, "_tool_schema"))
        
        self._tools_cache = tools
        return tools
    
    def has_tool(self, tool_name: str) -> bool:
        """检查该工具集是否包含指定工具"""
        for _, method in inspect.getmembers(self, inspect.ismethod):
            if hasattr(method, "_tool_name") and getattr(method, "_tool_name") == tool_name:
                return True
        return False
    
    async def invoke(self, tool_name: str, **kwargs) -> ToolResult:
        """根据工具名调用对应方法"""
        for _, method in inspect.getmembers(self, inspect.ismethod):
            if hasattr(method, "_tool_name") and getattr(method, "_tool_name") == tool_name:
                # 过滤 LLM 可能产生的幻觉参数
                filtered_kwargs = self._filter_parameters(method, kwargs)
                return await method(**filtered_kwargs)
        return ValueError(f"工具[{tool_name}]未找到")
    
    @classmethod
    def _filter_parameters(cls, method: Callable, kwargs: Dict) -> Dict:
        """只保留方法签名中存在的参数"""
        filtered = {}
        sign = inspect.signature(method)
        for key, value in kwargs.items():
            if key in sign.parameters:
                filtered[key] = value
        return filtered
```

### 2.3 使用示例：定义一个工具

```python
class MyTool(BaseTool):
    name: str = "my_toolset"
    
    @tool(
        name="do_something",
        description="做某事",
        parameters={
            "input": {"type": "string", "description": "输入内容"},
            "count": {"type": "integer", "description": "次数"},
        },
        required=["input"]
    )
    async def do_something(self, input: str, count: int = 1) -> ToolResult:
        result = input * count
        return ToolResult(success=True, data=result)
```

## 3. FileTool（文件工具）

这是使用最频繁的工具之一。Agent 需要读写文件来保存中间结果、分析数据、编写代码。

### 3.1 工具列表

```python
class FileTool(BaseTool):
    name: str = "file"
    
    @tool(name="file_read", description="读取文件内容", ...)
    async def file_read(self, filepath: str, start_line=None, end_line=None, sudo=False, max_length=10000)
    
    @tool(name="file_write", description="写入文件内容", ...)
    async def file_write(self, filepath: str, content: str, append=False, ...)
    
    @tool(name="file_str_replace", description="替换文件中的字符串", ...)
    async def file_str_replace(self, filepath: str, old_str: str, new_str: str, sudo=False)
    
    @tool(name="file_find_in_content", description="在文件内容中搜索", ...)
    async def file_find_in_content(self, filepath: str, regex: str, sudo=False)
    
    @tool(name="file_find_by_name", description="根据名称查找文件", ...)
    async def file_find_by_name(self, dir_path: str, glob_pattern: str)
    
    @tool(name="file_list", description="列出目录文件", ...)
    async def file_list(self, dir_path: str)
```

### 3.2 为什么用文件工具而非 Shell 命令？

系统提示词中明确要求：

> **必须使用文件工具进行读取、写入、追加和编辑，以避免 Shell 命令中出现的字符串转义问题**

Shell 中处理文件内容时，特殊字符（引号、换行、Unicode）的转义非常容易出错。文件工具通过 HTTP JSON API 传输，天然避免转义问题。

## 4. ShellTool（Shell 工具）

用于执行命令、运行代码、安装依赖。

### 4.1 工具列表

```python
class ShellTool(BaseTool):
    name: str = "shell"
    
    @tool(name="shell_execute", description="执行 Shell 命令", ...)
    async def shell_execute(self, session_id: str, exec_dir: str, command: str)
    
    @tool(name="shell_read_output", description="查看 Shell 输出", ...)
    async def shell_read_output(self, session_id: str)
    
    @tool(name="shell_wait_process", description="等待进程结束", ...)
    async def shell_wait_process(self, session_id: str, seconds: Optional[int] = None)
    
    @tool(name="shell_write_input", description="向进程写入输入", ...)
    async def shell_write_input(self, session_id: str, input_text: str, press_enter: bool)
    
    @tool(name="shell_kill_process", description="终止进程", ...)
    async def shell_kill_process(self, session_id: str)
```

### 4.2 Session ID 设计

Shell 命令通过 `session_id` 标识一个"会话"。同一个 session_id 的多次命令可以：
- 保持当前目录（cd 的效果）
- 保持环境变量
- 后台进程持续运行

这模拟了真实终端的体验。

### 5.2 可交互元素索引系统

这是浏览器工具的核心创新。PlaywrightBrowser 在提取页面内容时，会给所有可交互元素（按钮、链接、输入框）打上 `data-faber-id` 属性：

```javascript
// 页面中：
<button data-faber-id="faber-element-0">搜索</button>
<a data-faber-id="faber-element-1">新闻</a>
<input data-faber-id="faber-element-2" />
```

LLM 看到的内容：
```
0:<button>搜索</button>
1:<a>新闻</a>
2:<input />
```

LLM 只需要说"点击索引 0"，Agent 就能精确找到对应元素。这比让 LLM 生成 CSS Selector 或 XPath 更可靠（LLM 不擅长写精确的选择器）。

## 6. SearchTool（搜索工具）

```python
class SearchTool(BaseTool):
    name: str = "search"
    
    @tool(
        name="search_web",
        description="全网搜索引擎工具。当需要获取实时信息时使用。",
        parameters={
            "query": {"type": "string", "description": "优化后的查询字符串（3-5个关键词）"},
            "date_range": {
                "type": "string",
                "enum": ["all", "past_hour", "past_day", "past_week", "past_month", "past_year"],
                "description": "时间范围过滤"
            }
        },
        required=["query"]
    )
    async def search_web(self, query: str, date_range: Optional[str] = None) -> ToolResult[SearchResults]
```

**关键设计**：
- `query` 要求 LLM 提取关键词，而非使用完整问句（"北京 天气" 而非 "今天北京的天气怎么样"）
- `date_range` 枚举值让 LLM 可以选择时效性

## 7. MessageTool（消息工具）

这是"特殊"的工具——不操作外部环境，而是与用户交互。

```python
class MessageTool(BaseTool):
    name: str = "message"
    
    @tool(
        name="message_notify_user",
        description="向用户发送消息，且无需用户回复。用于进度更新。",
        parameters={"text": {"type": "string"}},
        required=["text"]
    )
    async def message_notify_user(self, text: str) -> ToolResult:
        return ToolResult(success=True, data="Continue")
    
    @tool(
        name="message_ask_user",
        description="向用户提问并等待回复。用于请求澄清或收集信息。",
        parameters={
            "text": {"type": "string"},
            "suggest_user_takeover": {"type": "string", "enum": ["none", "browser"]}
        },
        required=["text"]
    )
    async def message_ask_user(self, text: str, suggest_user_takeover=None) -> ToolResult:
        return ToolResult(success=True)
```

### 7.1 message_notify_user

ReActAgent 的 EXECUTION_PROMPT 中明确要求：

> **必须使用 `message_notify_user` 工具向用户通报进度，内容限制在一句话以内**

这让用户能实时了解 AI 在做什么：
- "正在使用搜索工具查询北京天气..."
- "已获取搜索结果，正在访问气象局官网..."

### 7.2 message_ask_user

当 Agent 需要用户输入时（如验证码、确认操作），调用此工具：

1. Agent 发送 `ToolEvent(CALLING)`，前端显示问题
2. Agent 发送 `ToolEvent(CALLED)`
3. ReActAgent 发送 `WaitEvent`
4. Flow 停止，会话状态变为 WAITING
5. 用户回复后，再次调用 `/chat`，Flow 继续执行

## 8. MCPTool（Model Context Protocol 工具）

MCP 是 Anthropic 提出的开放协议，用于标准化 AI 与外部工具的连接。

### 8.1 MCP 客户端管理器

```python
class MCPClientManager:
    def __init__(self, mcp_config: Optional[MCPConfig] = None):
        self._mcp_config = mcp_config
        self._exit_stack: AsyncExitStack = AsyncExitStack()
        self._clients: Dict[str, ClientSession] = {}  # 会话缓存
        self._tools: Dict[str, List[Tool]] = {}       # 工具 Schema 缓存
        self._initialized: bool = False
```

### 8.2 支持三种传输协议

```python
async def _connect_mcp_server(self, server_name: str, server_config: MCPServerConfig):
    transport = server_config.transport
    
    if transport == MCPTransport.STDIO:
        await self._connect_stdio_server(server_name, server_config)
    elif transport == MCPTransport.SSE:
        await self._connect_sse_server(server_name, server_config)
    elif transport == MCPTransport.STREAMABLE_HTTP:
        await self._connect_streamable_http_server(server_name, server_config)
```

**stdio 模式**：启动本地子进程（如 Python 脚本），通过 stdin/stdout 通信  
**sse 模式**：通过 HTTP SSE 长连接通信  
**streamable_http 模式**：通过流式 HTTP 通信（最新标准，支持双向流）

### 8.3 工具名前缀隔离

不同 MCP 服务可能有同名工具。系统通过前缀隔离：

```python
tool_name = f"mcp_{server_name}_{tool.name}"
# 例如：mcp_amap-maps-streamableHTTP_geocode
```

### 8.4 调用流程

1. Agent 请求调用 `mcp_amap_geocode`
2. MCPTool 解析出 server_name=`amap`、tool_name=`geocode`
3. 从 `_clients` 获取对应的 `ClientSession`
4. 调用 `session.call_tool("geocode", arguments)`
5. 将 MCP 的 `content` 列表转换为字符串返回

## 9. A2ATool（Agent-to-Agent 工具）

A2A 是 Google 提出的 Agent 间通信协议。

### 9.1 A2A 客户端管理器

```python
class A2AClientManager:
    async def initialize(self):
        # 1. 初始化 httpx 客户端
        self._httpx_client = await self._exit_stack.enter_async_context(
            httpx.AsyncClient(timeout=600)
        )
        
        # 2. 获取所有远程 Agent 的 Agent Card
        await self._get_a2a_agent_cards()
    
    async def _get_a2a_agent_cards(self):
        for config in self._a2a_config.a2a_servers:
            response = await self._httpx_client.get(
                f"{config.base_url}/.well-known/agent-card.json"
            )
            self._agent_cards[config.id] = response.json()
```

### 9.2 Agent Card

Agent Card 是 A2A 协议的标准发现机制，类似服务的 "README"，包含：
- Agent 名称、描述
- 支持的技能（skills）
- 调用端点 URL
- 认证方式

### 9.3 调用远程 Agent

```python
async def invoke(self, agent_id: str, query: str) -> ToolResult:
    agent_card = self._agent_cards[agent_id]
    url = agent_card["url"]
    
    response = await self._httpx_client.post(url, json={
        "id": str(uuid.uuid4()),
        "jsonrpc": "2.0",
        "method": "message/send",
        "params": {
            "message": {
                "messageId": str(uuid.uuid4()),
                "role": "user",
                "parts": [{"kind": "text", "text": query}],
            }
        }
    })
    
    return ToolResult(success=True, data=response.json())
```

## 10. 工具的初始化与清理

### 10.1 初始化时机

在 `AgentTaskRunner.invoke()` 中：

```python
async def invoke(self, task: Task) -> None:
    # 1. 确保沙箱就绪
    await self._sandbox.ensure_sandbox()
    
    # 2. 初始化 MCP 工具（连接所有配置的 MCP 服务器）
    await self._mcp_tool.initialize(self._mcp_config)
    
    # 3. 初始化 A2A 工具（获取远程 Agent Cards）
    await self._a2a_tool.initialize(self._a2a_config)
    
    # 4. 执行主循环...
```

### 10.2 清理的复杂性

MCP 和 A2A 的清理有一个 tricky 的问题：**anyio 的 cancel scope**。

`streamablehttp_client` 内部使用 `anyio.create_task_group()`，要求在**同一个 asyncio Task** 中进入和退出 cancel scope。如果 Task 被取消了（如用户停止会话），在另一个 Task 中清理会报错。

**解决方案**：在 `invoke()` 的 `finally` 块中清理（与初始化同一个 Task）：

```python
try:
    # 主循环...
finally:
    await self._cleanup_tools()  # 在同一个 Task 上下文
```

## 11. 工具系统的扩展指南

如果你想添加一个新工具，只需三步：

### 步骤 1：继承 BaseTool

```python
class CalculatorTool(BaseTool):
    name: str = "calculator"
```

### 步骤 2：用 @tool 装饰方法

```python
@tool(
    name="calculate",
    description="执行数学计算",
    parameters={
        "expression": {"type": "string", "description": "数学表达式，如 '2 + 2'"}
    },
    required=["expression"]
)
async def calculate(self, expression: str) -> ToolResult:
    try:
        result = eval(expression)  # 实际应用应使用更安全的方法
        return ToolResult(success=True, data=result)
    except Exception as e:
        return ToolResult(success=False, message=str(e))
```

### 步骤 3：注册到 Flow

在 `PlannerReActFlow.__init__()` 中将新工具加入列表：

```python
tools = [
    FileTool(sandbox=sandbox),
    ShellTool(sandbox=sandbox),
    BrowserTool(browser=browser),
    SearchTool(search_engine=search_engine),
    MessageTool(),
    CalculatorTool(),  # ← 新增
    mcp_tool,
    a2a_tool,
]
```

**无需其他配置**！`BaseAgent._get_available_tools()` 会自动通过反射发现新工具。
