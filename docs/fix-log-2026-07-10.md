## Changes

- 为 `BaseTool.invoke()` 增加统一超时保护，工具卡住时返回失败结果，避免前端长期停在 calling 状态。
- 为搜索工具设置更短超时时间，并缩短 Bing 搜索 HTTP 请求超时。
- 修复 ReAct 步骤结果解析：当模型输出 `[EARLY_COMPLETE]` 与 JSON 混合结构时，递归提取有效的 `success/result/attachments` 结果对象，避免 Pydantic 将 list 当作 `Step` 校验失败。
- 调整 Planner 更新计划逻辑：按步骤 id 更新已有步骤并追加新步骤，不再删除已经展示过的步骤，避免任务进度条缩水。
- 修复 `RedisStreamTask.destroy()` 遍历任务 registry 时被取消任务修改字典导致的 shutdown 异常。
- 补齐 memory 兼容层：

  - legacy `Memory.compact()`
  - lightweight `VectorMemory`
- 测试 fixture 自动注册/登录测试用户，并补充 VNC WebSocket token 兼容。
- 新增 ReAct early-complete 混合结构解析回归测试。
