# Faber API —— 核心业务流程：一次 Chat 请求的完整旅程

本章将追踪一个真实的用户请求——"帮我查一下北京今天的天气，并整理成一份 Markdown 文档"——从进入系统到最终返回的**完整链路**。

通过这个过程，你将理解各层如何协作，以及系统设计的精妙之处。

---

## 1. 阶段一：请求进入（接口层）

### 1.1 HTTP 请求到达

```
POST /api/sessions/{session_id}/chat
Content-Type: application/json

{
  "message": "帮我查一下北京今天的天气，并整理成一份 Markdown 文档",
  "attachments": [],
  "event_id": null,
  "timestamp": 1717845600
}
```

### 1.2 FastAPI 路由匹配

`session_routes.py` 中的 `chat()` 函数被调用：

```python
@router.post("/{session_id}/chat")
async def chat(
    session_id: str,
    request: ChatRequest,
    agent_service: AgentService = Depends(get_agent_service),
) -> EventSourceResponse:
```

**依赖注入**：FastAPI 调用 `get_agent_service()`，后者：
1. 读取 `config.yaml` 加载 LLMConfig、AgentConfig、MCPConfig
2. 创建 `OpenAILLM`、`OssFileStorage`、`RepairJSONParser`、`BingSearchEngine`
3. 组装成 `AgentService` 实例

### 1.3 SSE 响应准备

`chat()` 不直接返回数据，而是返回一个 `EventSourceResponse`，内部包含异步生成器 `event_generator()`。

FastAPI 开始保持 HTTP 连接打开，准备流式推送事件。

## 2. 阶段二：应用层调度（AgentService.chat）

### 2.1 获取会话

```python
async with self._uow:
    session = await self._uow.session.get_by_id(session_id)
```

数据库查询：
```sql
SELECT * FROM sessions WHERE id = 'xxx';
```

返回的 `SessionModel` 通过 `to_domain()` 转换为 `Session` 领域对象。

### 2.2 获取或创建任务

```python
task = await self._get_task(session)
```

检查 `session.task_id`：
- 如果存在 → 从全局注册表 `RedisStreamTask._task_registry` 获取已有 Task
- 如果不存在 → 返回 None

假设这是用户的第一条消息，`task` 为 None。

### 2.3 创建新任务（首次对话）

```python
if session.status != SessionStatus.RUNNING or task is None:
    task = await self._create_task(session)
```

#### 2.3.1 创建沙箱

```python
sandbox = await self._sandbox_cls.create()
```

`DockerSandbox.create()` 的执行过程：
1. 检查 `settings.sandbox_address` 是否配置
   - 如果配置 → 直接连接现有沙箱（开发调试模式）
   - 如果未配置 → 动态创建 Docker 容器
2. `docker.from_env().containers.run(...)` 启动容器
3. 从容器 `NetworkSettings` 提取 IP 地址
4. 返回 `DockerSandbox(ip=ip, container_name=name)`

此时，一个 Ubuntu 22.04 容器正在启动，内部 Chrome、VNC、HTTP API 服务正在初始化。

#### 2.3.2 获取浏览器

```python
browser = await sandbox.get_browser()  # → PlaywrightBrowser(cdp_url)
```

#### 2.3.3 创建 AgentTaskRunner

```python
task_runner = AgentTaskRunner(
    uow_factory=self._uow_factory,
    llm=self._llm,                    # OpenAILLM
    agent_config=self._agent_config,   # {max_iterations: 100, max_retries: 3}
    mcp_config=self._mcp_config,       # 配置的 MCP 服务列表
    a2a_config=self._a2a_config,       # 配置的 A2A 服务列表
    session_id=session.id,
    file_storage=self._file_storage,   # OssFileStorage
    json_parser=self._json_parser,     # RepairJSONParser
    browser=browser,                   # PlaywrightBrowser
    search_engine=self._search_engine, # BingSearchEngine
    sandbox=sandbox,                   # DockerSandbox
)
```

#### 2.3.4 创建 Task

```python
task = self._task_cls.create(task_runner=task_runner)  # → RedisStreamTask
```

RedisStreamTask 的初始化：
1. 生成 `task_id = uuid4()`
2. 创建输入 Stream：`task:input:{task_id}`
3. 创建输出 Stream：`task:output:{task_id}`
4. 注册到全局字典 `_task_registry[task_id] = self`

### 2.4 处理用户消息

```python
# 更新会话最新消息
async with self._uow:
    await self._uow.session.update_latest_message(session_id, message, timestamp)

# 创建用户消息事件
message_event = MessageEvent(role="user", message=message, attachments=[])

# 放入任务输入队列（Redis Stream）
event_id = await task.input_stream.put(message_event.model_dump_json())
message_event.id = event_id

# 先返回用户消息事件（前端立即显示用户发言）
yield message_event

# 持久化到数据库
async with self._uow:
    await self._uow.session.add_event(session_id, message_event)
```

### 2.5 启动任务执行

```python
await task.invoke()
```

RedisStreamTask.invoke()：
1. 检查 `self.done`（是否已有执行中的任务）
2. 创建后台 asyncio Task：`asyncio.create_task(self._execute_task())`
3. `_execute_task()` 调用 `task_runner.invoke(self)`

**此时，后台任务开始运行，而 `AgentService.chat()` 继续执行，进入事件消费循环。**

### 2.6 消费输出事件（流式返回）

```python
while task and not task.done:
    event_id, event_str = await task.output_stream.get(start_id=latest_event_id, block_ms=0)
    
    if event_str is None:
        continue  # 队列为空，继续轮询
    
    latest_event_id = event_id
    event = TypeAdapter(Event).validate_json(event_str)
    event.id = event_id
    
    # 重置未读消息数
    async with self._uow:
        await self._uow.session.update_unread_message_count(session_id, 0)
    
    yield event  # ← 流式推送给前端
    
    if isinstance(event, (DoneEvent, ErrorEvent, WaitEvent)):
        break
```

**关键设计**：`block_ms=0` 表示非阻塞。如果队列暂时为空，循环继续，不会卡住 FastAPI 协程。这样其他 HTTP 请求仍能被处理。

## 3. 阶段三：领域层执行（AgentTaskRunner）

### 3.1 初始化环境

```python
async def invoke(self, task: Task) -> None:
    await self._sandbox.ensure_sandbox()      # 等待沙箱就绪
    await self._mcp_tool.initialize(self._mcp_config)  # 连接 MCP 服务器
    await self._a2a_tool.initialize(self._a2a_config)  # 获取 A2A Agent Cards
```

`ensure_sandbox()` 轮询沙箱的 Supervisor 状态：
```
GET http://{sandbox_ip}:8080/api/supervisor/status
→ {services: [{name: "chrome", statename: "RUNNING"}, ...]}
```

### 3.2 主循环：读取输入、执行 Flow

```python
while not await task.input_stream.is_empty():
    event = await self._pop_event(task)  # 从输入队列弹出
    
    if isinstance(event, MessageEvent):
        message = event.message
        await self._sync_message_attachments_to_sandbox(event)
        
        message_obj = Message(message=message, attachments=[...])
        
        # 运行 PlannerReActFlow！
        async for event in self._run_flow(message_obj):
            await self._put_and_add_event(task, event)
            
            # 处理特殊事件
            if isinstance(event, TitleEvent):
                await self._uow.session.update_title(self._session_id, event.title)
            elif isinstance(event, MessageEvent):
                await self._uow.session.update_latest_message(...)
                await self._uow.session.increment_unread_message_count(...)
            elif isinstance(event, WaitEvent):
                await self._uow.session.update_status(self._session_id, SessionStatus.WAITING)
                return
```

### 3.3 Flow 执行：PlannerReActFlow.invoke()

#### 状态 1：PLANNING（生成计划）

```python
self.status = FlowStatus.PLANNING
async for event in self.planner.create_plan(message):
    if isinstance(event, PlanEvent) and event.status == PlanEventStatus.CREATED:
        self.plan = event.plan
        yield TitleEvent(title=event.plan.title)
        yield MessageEvent(role="assistant", message=event.plan.message)
    yield event
```

**实际 LLM 调用**：

PlannerAgent 调用 DeepSeek，携带 Planner 的系统提示词 + CREATE_PLAN_PROMPT。

LLM 返回：
```json
{
  "message": "好的，我来帮您查询北京今天的天气并整理成文档。",
  "language": "zh",
  "steps": [
    {"id": "1", "description": "使用搜索引擎查询北京今天的天气信息"},
    {"id": "2", "description": "访问权威天气网站获取详细数据"},
    {"id": "3", "description": "将天气信息整理成 Markdown 文档并保存到沙箱"}
  ],
  "goal": "获取北京今日天气并生成 Markdown 文档",
  "title": "北京天气查询与文档整理"
}
```

事件流：
1. `PlanEvent(status=CREATED, plan=Plan(...))` → 前端显示计划卡片
2. `TitleEvent(title="北京天气查询与文档整理")` → 前端更新会话标题
3. `MessageEvent(role="assistant", message="好的，我来帮您...")` → 前端显示 AI 回复

#### 状态 2：EXECUTING（执行步骤）

```python
self.status = FlowStatus.EXECUTING
step = self.plan.get_next_step()  # → Step 1: 搜索天气

async for event in self.react.execute_step(self.plan, step, message):
    yield event
```

**ReActAgent 执行 Step 1**：

1. **发送 StepEvent(STARTED)** → 前端显示"正在执行：使用搜索引擎查询北京今天的天气信息"

2. **调用 LLM**（携带 ReAct 系统提示词 + EXECUTION_PROMPT）：

   LLM 决定调用工具：
   ```json
   {
     "tool_calls": [{
       "id": "call_123",
       "function": {
         "name": "search_web",
         "arguments": "{\"query\": \"北京 天气\", \"date_range\": \"past_day\"}"
       }
     }]
   }
   ```

3. **BaseAgent 处理工具调用**：
   - 发送 `ToolEvent(CALLING, tool_name="search", function_name="search_web", ...)` → 前端显示"正在搜索..."
   - 调用 `SearchTool.search_web(query="北京 天气")`
   - BingSearchEngine 发起 HTTP 请求，获取搜索结果
   - 发送 `ToolEvent(CALLED, ..., function_result=ToolResult(data=SearchResults(...)))` → 前端显示搜索结果列表

4. **将工具结果喂回 LLM**：

   LLM 看到搜索结果后，决定下一步：
   ```json
   {
     "tool_calls": [{
       "function": {
         "name": "browser_navigate",
         "arguments": "{\"url\": \"https://weather.com/zh-CN/weather/today/l/CHXX0008:1:CH\"}"
       }
     }]
   }
   ```

5. **浏览器操作**：
   - `ToolEvent(CALLING, tool_name="browser", function_name="browser_navigate")` → 前端显示"正在访问天气网站..."
   - PlaywrightBrowser 通过 CDP 控制沙箱中的 Chrome，导航到目标 URL
   - `ToolEvent(CALLED, ..., tool_content=BrowserToolContent(screenshot="..."))` → 前端显示网页截图

6. **继续迭代**：

   LLM 继续决定下一步（可能是 `browser_view` 查看页面内容，或 `file_write` 写入文件）。ReAct 循环最多执行 `max_iterations`（默认 100）次。

7. **步骤完成**：

   当 LLM 不再调用工具，而是返回 JSON 结果：
   ```json
   {
     "success": true,
     "result": "已成功获取北京今日天气：晴，25°C，北风 3 级。",
     "attachments": []
   }
   ```

   ReActAgent：
   - 发送 `StepEvent(COMPLETED, step=Step(id="1", ...))` → 前端标记步骤完成
   - 发送 `MessageEvent(role="assistant", message="已成功获取北京今日天气...")` → 前端显示结果

8. **记忆压缩**：
   ```python
   await self.react.compact_memory()
   ```
   删除浏览器页面源码、思考过程，释放 Token。

#### 状态 3：UPDATING（更新计划）

```python
self.status = FlowStatus.UPDATING
async for event in self.planner.update_plan(self.plan, step):
    yield event
```

PlannerAgent 看到 Step 1 已完成，更新剩余步骤。可能输出：
```json
{
  "steps": [
    {"id": "2", "description": "访问权威天气网站获取详细数据"},
    {"id": "3", "description": "将天气信息整理成 Markdown 文档并保存到沙箱"}
  ]
}
```

Flow 状态回到 EXECUTING，继续执行 Step 2。

#### 循环执行...

Step 2 和 Step 3 重复上述过程，直到所有步骤完成。

#### 状态 4：SUMMARIZING（总结）

```python
self.status = FlowStatus.SUMMARIZING
async for event in self.react.summarize():
    yield event
```

ReActAgent 汇总所有步骤的结果，生成最终回复：
```json
{
  "message": "已为您完成北京天气查询。今日北京天气晴，最高气温 28°C，最低气温 18°C，北风 3-4 级，空气质量优。详细报告已保存到 /home/ubuntu/weather_report.md。",
  "attachments": ["/home/ubuntu/weather_report.md"]
}
```

AgentTaskRunner 将附件同步到 OSS，生成访问 URL。

#### 状态 5：COMPLETED（完成）

```python
self.status = FlowStatus.COMPLETED
yield PlanEvent(status=COMPLETED, plan=self.plan)
yield DoneEvent()
```

AgentTaskRunner 更新会话状态为 COMPLETED。

## 5. 阶段五：请求结束

### 5.1 正常结束

所有事件推送完毕，FastAPI 关闭 SSE 连接。`AgentService.chat()` 的 `finally` 块执行：

```python
finally:
    asyncio.create_task(self._safe_update_unread_count(session_id))
```

创建后台任务清零未读消息数（避免被 SSE 断连的 cancel scope 中断）。

### 5.2 用户中途打断

假设用户在 Step 2 执行时发送了新消息：

1. 前端再次调用 `POST /sessions/{session_id}/chat`
2. `AgentService.chat()` 发现 `session.status == RUNNING`
3. 获取已有 Task（不是创建新 Task）
4. 新消息放入 Task 的输入队列
5. `AgentTaskRunner` 的主循环检测到输入队列非空，跳出当前步骤执行
6. `PlannerReActFlow` 检测到 `session.status == RUNNING`，状态设为 PLANNING，重新规划
7. `PlannerAgent.roll_back()` 和 `ReActAgent.roll_back()` 清理记忆，确保消息列表格式正确
8. 基于新消息生成新计划，继续执行

### 5.3 用户停止任务

用户点击"停止"按钮：

1. 前端调用 `POST /sessions/{session_id}/stop`
2. `AgentService.stop_session(session_id)` 调用 `task.cancel()`
3. `RedisStreamTask.cancel()` 取消后台 asyncio Task
4. `AgentTaskRunner` 的 `invoke()` 捕获 `asyncio.CancelledError`
5. 发送 `DoneEvent()`，更新会话状态为 COMPLETED
6. `finally` 块清理 MCP/A2A 资源

## 7. 关键设计洞察

### 7.1 异步生成器的链式传递

从领域层到接口层，事件通过**异步生成器链**传递：

```python
# 领域层
async def invoke(self) -> AsyncGenerator[BaseEvent, None]:
    yield PlanEvent(...)
    yield ToolEvent(...)
    yield DoneEvent()

# 应用层
async def chat(self) -> AsyncGenerator[BaseEvent, None]:
    async for event in self._run_flow(message):
        yield event

# 接口层
async def event_generator() -> AsyncGenerator[ServerSentEvent, None]:
    async for event in agent_service.chat():
        sse_event = EventMapper.event_to_sse_event(event)
        yield ServerSentEvent(event=sse_event.event, data=sse_event.data)
```

这种设计让每一层都可以**透明地拦截和转换**事件，而无需累积所有事件到内存中。

### 7.2 Redis Stream 作为"时间旅行"机制

Redis Stream 的消息 ID 是时间戳+序列号（如 `1717845600000-0`）。前端传入 `event_id` 参数后，系统可以从该 ID 之后开始读取：

```python
await task.output_stream.get(start_id=latest_event_id, block_ms=0)
```

这意味着：
- **断线重连**：前端断开 SSE 后重新连接，传入最后收到的事件 ID，可以无缝续接
- **历史回放**：新加入的观察者可以从 Stream 开头读取所有历史事件

### 7.3 Task 与 Session 的解耦

- **Session** 是持久化的数据库实体，长期存在
- **Task** 是内存中的执行单元，随对话创建和销毁

一个 Session 可以有多个 Task（多次对话），但共享同一个沙箱。这种设计：
- 保留了对话历史（在 Session.events 中）
- 允许随时中断和重新开始
- 沙箱状态可以跨 Task 保留（如之前创建的文件）
