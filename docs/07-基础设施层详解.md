# Faber API —— 基础设施层（Infrastructure）详解

基础设施层是系统的"四肢"，负责所有与技术相关的实现：数据库访问、Redis 操作、HTTP 请求、Docker 管理、浏览器控制等。

**核心原则**：基础设施层**实现领域层定义的协议**，但不影响领域层的业务逻辑。

## 1. 存储（Storage）

### 1.1 PostgreSQL（postgres.py）

```python
class Postgres:
    def __init__(self):
        self._engine: Optional[AsyncEngine] = None
        self._session_factory: Optional[async_sessionmaker] = None
        self._settings = get_settings()
    
    async def init(self) -> None:
        if self._engine is not None:
            return
        
        self._engine = create_async_engine(
            self._settings.sqlalchemy_database_uri,
            echo=True if self._settings.env == "development" else False,
            pool_pre_ping=True,  # 关键！防止使用已关闭的连接
        )
        
        self._session_factory = async_sessionmaker(
            autocommit=False,
            autoflush=False,
            bind=self._engine,
        )
        
        # 安装 uuid-ossp 扩展
        async with self._engine.begin() as conn:
            await conn.execute(text('CREATE EXTENSION IF NOT EXISTS "uuid-ossp";'))
    
    async def shutdown(self) -> None:
        if self._engine:
            await self._engine.dispose()
            get_postgres.cache_clear()
```

**关键配置**：
- `pool_pre_ping=True`：从连接池取出连接前，先发送一个轻量级 ping。如果连接已断开，自动创建新连接。这是生产环境防止 "connection closed unexpectedly" 的必备配置。
- `echo=True`（仅开发模式）：打印所有 SQL 语句，便于调试。

**单例模式**：

```python
@lru_cache()
def get_postgres() -> Postgres:
    return Postgres()
```

`@lru_cache` 确保整个进程只有一个 Postgres 实例。

### 1.2 Redis（redis.py）

```python
class RedisClient:
    def __init__(self):
        self._client: Redis | None = None
        self._settings = get_settings()

            host=self._settings.redis_host,

            decode_responses=True,  # 自动将字节解码为字符串

    def client(self) -> Redis:
```

**decode_responses=True**：Redis 默认返回字节，`decode_responses=True` 让客户端自动用 UTF-8 解码，避免手动 `decode()`。

## 2. ORM 模型与仓库实现

### 2.1 ORM 模型（SessionModel）

```python
class SessionModel(Base):
    __tablename__ = "sessions"
    
    id: Mapped[str] = mapped_column(String(255), primary_key=True)
    events: Mapped[List[Dict]] = mapped_column(JSONB, server_default=text("'[]'::jsonb"))
    memories: Mapped[Dict] = mapped_column(JSONB, server_default=text("'{}'::jsonb"))
    files: Mapped[List[Dict]] = mapped_column(JSONB, server_default=text("'[]'::jsonb"))
    status: Mapped[str] = mapped_column(String(255), server_default=text("''::character varying"))
```

**SQLAlchemy 2.0 新特性**：
- `Mapped[T]` 类型注解：IDE 可以提供类型提示
- `mapped_column()`：替代旧的 `Column()`，更类型安全

### 2.2 领域模型与 ORM 模型的转换

```python
class SessionModel(Base):
    @classmethod
    def from_domain(cls, session: Session) -> "SessionModel":
        return cls(
            **session.model_dump(
                mode="python",
                exclude={"memories", "files", "events", "updated_at", "created_at"},
            ),
            **session.model_dump(
                mode="json",
                include={"memories", "files", "events"},
            )
        )
    
    def to_domain(self) -> Session:
        return Session.model_validate(self, from_attributes=True)
```

**mode="python" vs mode="json"**：
- `mode="python"`：保留 Python 原生类型（如 `datetime`、`UUID`）
- `mode="json"`：序列化为 JSON 兼容类型（如 `datetime` → ISO 字符串）

对于 PostgreSQL JSONB 字段，需要用 `mode="json"` 确保数据可序列化。

### 2.3 DBSessionRepository

```python
class DBSessionRepository(SessionRepository):
    def __init__(self, db_session: AsyncSession):
        self.db_session = db_session
    
    async def save(self, session: Session) -> None:
        stmt = select(SessionModel).where(SessionModel.id == session.id)
        result = await self.db_session.execute(stmt)
        record = result.scalar_one_or_none()
        
        if not record:
            record = SessionModel.from_domain(session)
            self.db_session.add(record)
        else:
            record.update_from_domain(session)
    
    async def add_event(self, session_id: str, event: BaseEvent) -> None:
        event_data = event.model_dump(mode="json")
        stmt = (
            update(SessionModel)
            .where(SessionModel.id == session_id)
            .values(
                events=func.coalesce(SessionModel.events, cast([], JSONB)) + cast([event_data], JSONB),
            )
        )
        await self.db_session.execute(stmt)
```

**原子性事件追加**：

```sql
UPDATE sessions 
SET events = COALESCE(events, '[]'::jsonb) || '[{"type": "message", ...}]'::jsonb
WHERE id = 'xxx';
```

使用 PostgreSQL 的 `||` 运算符直接追加 JSONB 数组，无需先读取整行。

### 2.4 DBUnitOfWork

```python
class DBUnitOfWork(IUnitOfWork):
    def __init__(self, session_factory: async_sessionmaker[AsyncSession]):
        self.session_factory = session_factory
        self.db_session: Optional[AsyncSession] = None
    
    async def __aenter__(self) -> "DBUnitOfWork":
        self.db_session = self.session_factory()
        self.file = DBFileRepository(db_session=self.db_session)
        self.session = DBSessionRepository(db_session=self.db_session)
        return self
    
    async def __aexit__(self, exc_type, exc_val, exc_tb):
        try:
            if exc_type:
                await self.rollback()
            else:
                await self.commit()
        except asyncio.CancelledError:
            logger.warning("UoW 提交/回滚操作被取消(可能是客户端断开连接)")
        except Exception as e:
            logger.warning(f"UoW 提交/回滚操作失败: {e}")
        finally:
            try:
                await self.db_session.close()
            except asyncio.CancelledError:
                logger.warning("UoW 关闭数据库会话被取消")
            except Exception as e:
                logger.warning(f"UoW 关闭数据库会话失败: {e}")
```

**关键设计：防御性异常处理**

当 SSE 客户端断开时，`asyncio.CancelledError` 会传播到 `__aexit__`。如果直接抛出：
1. `commit()` 被中断 → 数据库连接处于不确定状态
2. `close()` 也被跳过 → 连接泄漏

解决方案：**在 `__aexit__` 的每个阶段都捕获 `CancelledError`**，记录日志但不抛出，确保 `close()` 一定被执行。

## 3. LLM 实现（OpenAILLM）

```python
class OpenAILLM(LLM):
    def __init__(self, llm_config: LLMConfig, **kwargs):
        self._client = AsyncOpenAI(
            base_url=str(llm_config.base_url),
            api_key=llm_config.api_key,
            **kwargs,
        )
        self._model_name = llm_config.model_name
        self._temperature = llm_config.temperature
        self._max_tokens = llm_config.max_tokens
        self._timeout = 3600  # 1 小时超时（长文本生成可能很慢）
    
    async def invoke(self, messages, tools=None, response_format=None, tool_choice=None):
        if tools:
            response = await self._client.chat.completions.create(
                model=self._model_name,
                temperature=self._temperature,
                max_tokens=self._max_tokens,
                messages=messages,
                response_format=response_format,
                tools=tools,
                tool_choice=tool_choice,
                parallel_tool_calls=False,  # 关键！关闭并行工具调用
                timeout=self._timeout,
            )
        else:
            response = await self._client.chat.completions.create(
                model=self._model_name,
                ...
            )
        
        return response.choices[0].message.model_dump()
```

**parallel_tool_calls=False**：

OpenAI 的 GPT-4 支持一次调用多个工具（如同时搜索和读取文件）。但 DeepSeek 和部分模型不支持此参数，且并行调用的结果汇总逻辑复杂。项目明确关闭此功能，强制**一次只调用一个工具**。

## 4. Docker 沙箱（DockerSandbox）

这是基础设施层**最复杂**的实现，涉及 Docker SDK、HTTP 客户端、容器网络等多个技术点。

### 4.1 核心设计

DockerSandbox 不直接在代码中执行 Shell 命令，而是**通过 HTTP API 与沙箱容器通信**：

```
Faber API 后端 ──HTTP──▶ 沙箱容器:8080
                         ├─ /api/file/read-file
                         ├─ /api/file/write-file
                         ├─ /api/shell/exec-command
                         ├─ /api/supervisor/status
                         └─ ...
```

沙箱容器内部运行一个轻量级 HTTP 服务（基于 Supervisor 管理），暴露文件、Shell、进程等操作。

### 4.2 创建沙箱

```python
@classmethod
def _create_task(cls) -> Self:
    settings = get_settings()
    container_name = f"{settings.sandbox_name_prefix}-{str(uuid.uuid4())[:8]}"
    
    docker_client = docker.from_env()
    container = docker_client.containers.run(
        image=settings.sandbox_image,
        name=container_name,
        detach=True,
        remove=True,  # 停止后自动删除容器
        environment={
            "SERVICE_TIMEOUT_MINUTES": settings.sandbox_ttl_minutes,
            "CHROME_ARGS": settings.sandbox_chrome_args,
            "HTTPS_PROXY": settings.sandbox_https_proxy,
        },
        network=settings.sandbox_network,
    )
    
    container.reload()
    ip = cls._get_container_ip(container)
    
    return DockerSandbox(ip=ip, container_name=container_name)
```

**remove=True**：容器停止后自动删除，避免磁盘空间泄漏。

### 4.3 获取已有沙箱

```python
@classmethod
async def get(cls, id: str) -> Optional[Self]:
    docker_client = docker.from_env()
    try:
        container = docker_client.containers.get(id)
        container.reload()
        
        if container.status != "running":
            return None
        
        ip = cls._get_container_ip(container)
        return DockerSandbox(ip=ip, container_name=id)
    except NotFound:
        return None  # 容器已被销毁
    finally:
        docker_client.close()
```

### 4.4 IP 地址解析

```python
@classmethod
@alru_cache(maxsize=128, typed=True)
async def _resolve_hostname_to_ip(cls, hostname: str) -> Optional[str]:
    try:
        # 先判断是不是已经是 IP
        socket.inet_pton(socket.AF_INET, hostname)
        return hostname
    except OSError:
        pass
    
    # 通过 DNS 解析
    addr_info = socket.getaddrinfo(hostname, None, family=socket.AF_INET)
    if addr_info:
        return addr_info[0][4][0]
    return None
```

**@alru_cache**：异步 LRU 缓存，避免重复解析同一 hostname。

### 4.5 确保沙箱就绪

```python
async def ensure_sandbox(self) -> None:
    max_retries = 30
    retry_interval = 2
    
    for attempt in range(max_retries):
        response = await self.client.get(f"{self._base_url}/api/supervisor/status")
        tool_result = ToolResult.from_sandbox(**response.json())
        
        services = tool_result.data or []
        all_running = all(
            service.get("statename") == "RUNNING"
            for service in services
        )
        
        if all_running:
            return
        
        await asyncio.sleep(retry_interval)
    
    raise Exception("沙箱服务启动超时")
```

沙箱容器启动后，内部的服务（Chrome、VNC、HTTP API）需要时间初始化。`ensure_sandbox()` 轮询 Supervisor 状态，最多等待 60 秒。

```python
async def upload_file(self, file_data: BinaryIO, filepath: str, filename: str = None):
```

## 5. Playwright 浏览器（PlaywrightBrowser）

### 5.1 CDP 远程连接

与常规 Playwright 用法（本地启动浏览器）不同，Faber **远程连接**沙箱中已运行的 Chrome：

```python
async def initialize(self) -> bool:
    self.playwright = await async_playwright().start()
    self.browser = await self.playwright.chromium.connect_over_cdp(self.cdp_url)
    
    contexts = self.browser.contexts
    if contexts and len(contexts[0].pages) == 1:
        page = contexts[0].pages[0]
        # 如果是空白页，直接使用；否则新建页面
        if page.url in ["about:blank", "chrome://newtab/"]:
            self.page = page
        else:
            self.page = await contexts[0].new_page()
    else:
        context = contexts[0] if contexts else await self.browser.new_context()
        self.page = await context.new_page()
    
    return True
```

**connect_over_cdp(url)**：通过 Chrome DevTools Protocol 连接已有浏览器实例。这是实现"远程浏览器控制"的关键。

### 5.2 页面内容提取

```python
async def _extract_content(self) -> str:
```

**markdownify**：将 HTML 转换为 Markdown，LLM 对 Markdown 的理解远好于原始 HTML。

### 5.4 元素点击

```python
async def click(self, index=None, coordinate_x=None, coordinate_y=None):
    await self._ensure_page()
    
    if coordinate_x is not None and coordinate_y is not None:
        await self.page.mouse.click(coordinate_x, coordinate_y)
    elif index is not None:
        element = await self._get_element_by_id(index)
```

- 如果元素不可见，自动滚动到视口
- 5 秒点击超时
- 支持坐标点击作为备选方案

## 6. Redis Stream 任务队列

### 6.1 RedisStreamMessageQueue

```python
class RedisStreamMessageQueue(MessageQueue):
    def __init__(self, stream_name: str):
        self._stream_name = stream_name
        self._redis = get_redis()
        self._lock_expire_seconds = 10
    
    async def put(self, message: Any) -> str:
        return await self._redis.client.xadd(self._stream_name, {"data": message})
    
    async def get(self, start_id: str = None, block_ms: int = None) -> Tuple[str, Any]:
        if start_id is None:
            start_id = '0'
        
        messages = await self._redis.client.xread(
            {self._stream_name: start_id},
            count=1,
            block=block_ms,
        )
        
        if not messages:
            return None, None
        
        stream_messages = messages[0][1]
        message_id, message_data = stream_messages[0]
        return message_id, message_data.get("data")
    
    async def pop(self) -> Tuple[str, Any]:
        # 使用分布式锁保证原子性
        lock_key = f"lock:{self._stream_name}:pop"
        lock_value = await self._acquire_lock(lock_key)
        
        try:
            messages = await self._redis.client.xrange(self._stream_name, "-", "+", count=1)
            if not messages:
                return None, None
            
            message_id, message_data = messages[0]
            await self._redis.client.xdel(self._stream_name, message_id)
            return message_id, message_data.get("data")
        finally:
            await self._release_lock(lock_key, lock_value)
```

**为什么用 Redis Stream 而非 List？**

| 特性 | Redis List | Redis Stream |
|------|-----------|--------------|
| 消息 ID | 无（只有索引） | 自动生成时间戳+序列号 |
| 消费者组 | 不支持 | 原生支持 |
| 消息持久化 | 弹出即消失 | 可保留历史 |
| 阻塞读取 | `BLPOP` | `XREAD BLOCK` |

Stream 更适合需要"从指定位置开始读取"的场景（如 SSE 断线重连）。

### 6.2 分布式锁

```python
async def _acquire_lock(self, lock_key: str, timeout_seconds: int = 5) -> Optional[str]:
    lock_value = str(uuid.uuid4())
    end_time = timeout_seconds
    
    while end_time > 0:
        result = await self._redis.client.set(
            lock_key,
            lock_value,
            nx=True,           # 仅当 key 不存在时才设置
            ex=self._lock_expire_seconds,
        )
        if result:
            return lock_value
        
        await asyncio.sleep(0.1)
        end_time -= 0.1
    
    return None

async def _release_lock(self, lock_key: str, lock_value: str) -> bool:
    # 使用 Lua 脚本保证原子性
    release_script = """
    if redis.call("GET", KEYS[1]) == ARGV[1] then
        return redis.call("DEL", KEYS[1])
    else
        return 0
    end
    """
    script = self._redis.client.register_script(release_script)
    result = await script(keys=[lock_key], args=[lock_value])
    return result == 1
```

**Lua 脚本的必要性**："先 GET 再 DEL" 不是原子操作，在高并发下可能误删其他客户端的锁。Lua 脚本在 Redis 服务端原子执行。

### 6.3 RedisStreamTask

```python
class RedisStreamTask(Task):
    _task_registry: Dict[str, "RedisStreamTask"] = {}
    
    def __init__(self, task_runner: TaskRunner):
        self._task_runner = task_runner
        self._id = str(uuid.uuid4())
        self._execution_task: Optional[asyncio.Task] = None
        
        self._input_stream = RedisStreamMessageQueue(f"task:input:{self._id}")
        self._output_stream = RedisStreamMessageQueue(f"task:output:{self._id}")
        
        RedisStreamTask._task_registry[self._id] = self
    
    async def invoke(self) -> None:
        if self.done:
            self._execution_task = asyncio.create_task(self._execute_task())
    
    async def _execute_task(self) -> None:
        try:
            await self._task_runner.invoke(self)
        except asyncio.CancelledError:
            logger.info(f"任务[{self._id}]执行被取消")
            raise
        except Exception as e:
            logger.error(f"任务[{self._id}]执行出现异常: {e}")
        finally:
            self._on_task_done()
    
    def cancel(self) -> bool:
        if not self.done:
            self._execution_task.cancel()
            self._cleanup_registry()
            return True
        return True
    
    @classmethod
    def get(cls, task_id: str) -> Optional["Task"]:
        return RedisStreamTask._task_registry.get(task_id)
```

**全局注册表**：`_task_registry` 是一个类级字典，保存所有活跃任务的引用。这样 AgentService 可以通过 `task_id` 快速找到已有任务，无需查询数据库。

## 7. 文件存储（OSS / COS）

### 7.1 OssFileStorage

```python
class OssFileStorage(FileStorage):
    def __init__(self, bucket: str, oss: Oss, uow_factory: Callable):
        self._bucket = bucket
        self._oss = oss
        self._uow_factory = uow_factory
    
    async def upload_file(self, upload_file: UploadFile) -> File:
        file_id = str(uuid.uuid4())
        key = f"uploads/{file_id}/{upload_file.filename}"
        
        # 读取文件内容
        content = await upload_file.read()
        
        # 上传到 OSS
        self._oss.bucket.put_object(key, content)
        
        # 创建 File 记录
        file = File(
            id=file_id,
            filename=upload_file.filename,
            key=key,
            url=f"https://{self._bucket}.{self._oss.endpoint}/{key}",
            size=len(content),
            mime_type=upload_file.content_type or "application/octet-stream",
        )
        
        # 持久化到数据库
        uow = self._uow_factory()
        async with uow:
            await uow.file.save(file)
        
        return file
```

### 7.2 文件同步流程

用户上传文件到 OSS → 用户发消息附带 File ID → AgentService 下载文件 → 上传到沙箱 → AI 在沙箱中操作文件 → AI 生成新文件 → 新文件上传回 OSS → 前端通过 URL 访问。

## 8. 健康检查

### 8.1 PostgresHealthChecker

```python
class PostgresHealthChecker(HealthChecker):
    def __init__(self, db_session: AsyncSession):
        self._db_session = db_session
    
    async def check(self) -> HealthStatus:
        try:
            await self._db_session.execute(text("SELECT 1"))
            return HealthStatus(name="postgres", status="healthy")
        except Exception as e:
            return HealthStatus(name="postgres", status="error", message=str(e))
```

### 8.2 RedisHealthChecker

```python
class RedisHealthChecker(HealthChecker):
    def __init__(self, redis_client: RedisClient):
        self._redis_client = redis_client
    
    async def check(self) -> HealthStatus:
        try:
            await self._redis_client.client.ping()
            return HealthStatus(name="redis", status="healthy")
        except Exception as e:
            return HealthStatus(name="redis", status="error", message=str(e))
```
