## 1. `BaseTool` 基类

文件：`app/domain/services/tools/base.py:49`

```python
class BaseTool:
    name: str = ""  # 工具集的名字

    def __init__(self) -> None:
        self._tools_cache = None

    def get_tools(self) -> List[Dict[str, Any]]:
        """获取所有已注册的工具 schema"""

    def has_tool(self, tool_name: str) -> bool:
        """判断工具集中是否存在某个工具"""

    async def invoke(self, tool_name: str, **kwargs) -> ToolResult:
        """根据工具名调用对应方法"""
```

### 1.1 核心职责

- **统一入口**：所有工具通过 `invoke(tool_name, **kwargs)` 被外部调用
- **Schema 生成**：`get_tools()` 用反射扫描带 `_tool_schema` 属性的方法，生成 OpenAI 格式的 function schema 列表
- **参数过滤**：`_filter_parameters()` 会剔除 LLM 幻觉产生的多余参数，只保留方法签名中存在的参数

### 1.2 `get_tools()` 实现

```python
def get_tools(self) -> List[Dict[str, Any]]:
    if self._tools_cache is not None:
        return self._tools_cache

    tools = []
    for _, method in inspect.getmembers(self, inspect.ismethod):
        if hasattr(method, "_tool_schema"):
            tools.append(getattr(method, "_tool_schema"))

    self._tools_cache = tools
    return tools
```

通过 `inspect.getmembers` 遍历实例方法，找到被 `@tool` 装饰过的方法，把它们的 `_tool_schema` 收集起来。结果被缓存，避免重复反射。

### 1.3 `invoke()` 实现

```python
async def invoke(self, tool_name: str, **kwargs) -> ToolResult:
    for _, method in inspect.getmembers(self, inspect.ismethod):
        if hasattr(method, "_tool_name") and getattr(method, "_tool_name") == tool_name:
            filtered_kwargs = self._filter_parameters(method, kwargs)
            return await method(**filtered_kwargs)
    return ValueError(f"工具[{tool_name}]未找到")
```

调用路径：
1. 按 `tool_name` 匹配方法
2. 用 `_filter_parameters()` 过滤参数
3. 执行方法并返回 `ToolResult`

### 1.4 `_filter_parameters()` 实现

```python
@classmethod
def _filter_parameters(cls, method: Callable, kwargs: Dict[str, Any]) -> Dict[str, Any]:
    filtered_kwargs = {}
    sign = inspect.signature(method)
    for key, value in kwargs.items():
        if key in sign.parameters:
            filtered_kwargs[key] = value
    return filtered_kwargs
```

这是一个防御性设计。LLM 生成工具参数时可能产生幻觉（比如传入方法签名里没有的参数），通过 `inspect.signature` 过滤后只保留合法参数，避免调用报错。

## 2. `@tool` 装饰器

文件：`app/domain/services/tools/base.py:15`

```python
def tool(
    name: str,
    description: str,
    parameters: Dict[str, Dict[str, Any]],
    required: List[str],
) -> Callable:
    def decorator(func):
        tool_schema = {
            "type": "function",
            "function": {
                "name": name,
                "description": description,
                "parameters": {
                    "type": "object",
                    "properties": parameters,
                    "required": required,
                }
            }
        }
        func._tool_name = name
        func._tool_description = description
        func._tool_schema = tool_schema
        return func
    return decorator
```

### 2.1 作用

被 `@tool` 装饰的方法会挂载三个属性：

| 属性 | 说明 |
|---|---|
| `_tool_name` | 工具名，LLM 调用时使用的标识 |
| `_tool_description` | 工具描述，帮助 LLM 理解用途 |
| `_tool_schema` | 完整的 OpenAI function schema |

### 2.2 使用示例

```python
class ShellTool(BaseTool):
    @tool(
        name="shell_execute",
        description="执行shell命令",
        parameters={
            "command": {"type": "string", "description": "要执行的命令"},
        },
        required=["command"],
    )
    async def shell_execute(self, command: str) -> ToolResult:
        ...
```

## 5. 工具调用链

文件：`app/domain/services/agents/base.py:273-344`

```python
async def invoke(self, ...):
    while True:
        response = await self.llm.invoke(messages, tools=self.tool_schemas)
        if response.tool_calls:
            tool_call = response.tool_calls[0]  # 每次只取第一个
            yield ToolEvent(status=CALLING, ...)
            result = await self._invoke_tool(tool_call)
            yield ToolEvent(status=CALLED, function_result=result, ...)
```

完整链路：
1. LLM 根据 `tool_schemas` 决定调用哪个工具
2. `BaseAgent._invoke_tool()` 找到对应 `BaseTool` 实例
3. 调用 `BaseTool.invoke(tool_name, **kwargs)`
4. 实际方法执行，返回 `ToolResult`
5. 结果写回 messages，进入下一轮思考/行动

## 6. 工具事件

文件：`app/domain/models/event.py`

每次工具调用前后都会产生 `ToolEvent`：

```python
class ToolEvent(BaseEvent):
    tool_name: str
    function_name: str
    function_args: Dict[str, Any]
    function_result: Optional[ToolResult] = None
    status: ToolEventStatus  # CALLING / CALLED
    tool_content: Optional[BrowserToolContent] = None
```

- `CALLING`：工具即将执行
- `CALLED`：工具执行完成，携带 `function_result`

浏览器工具还会额外附加 `BrowserToolContent` 截图 URL。

## 7. 设计特点总结

1. **统一抽象**：所有工具继承 `BaseTool`，返回 `ToolResult`
2. **声明式注册**：`@tool` 装饰器把方法变成 LLM 可调用的 function schema
3. **参数安全**：`_filter_parameters()` 过滤 LLM 幻觉参数
4. **缓存优化**：`get_tools()` 缓存 schema，避免重复反射
5. **事件驱动**：工具调用与事件流结合，支持 SSE 实时推送到前端
